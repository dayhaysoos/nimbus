limits.

Worker invocation. That invocation synchronously waits on `nimbus-agent-endpoint` 
which synchronously waits on OpenRouter. Large reviews with slow models get 
canceled because the Worker wall time ceiling is hit while waiting for the model 
response.


The fix: move `processReviewRun` execution into a new `ReviewRunner` Durable Object. 
DOs hibernate during I/O waits and are not subject to the same wall time constraints 
as Worker invocations.

   - Owns the full `processReviewRun` lifecycle
   - Receives a review job payload via a fetch call from the queue consumer
   - Runs context assembly, co-change lookup, sandbox operations, and model call 
     inside the DO
   - Uses `this.ctx.waitUntil` or alarm-based wake for long I/O waits where 
     appropriate
   - Persists its own execution state in DO storage so it survives hibernation

     `reviewId`

   - The DO runs independently from this point
