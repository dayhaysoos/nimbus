nimbus review create --commit <commit-ish>

- On invocation, resolve the Entire checkpoint trailer (`Entire-Checkpoint:`) from the specified commit. If no checkpoint trailer is found, hard-fail immediately with a clear error — no checkpoint means no review, consistent with existing hard constraints.
- Internally orchestrate workspace creation → deployment → review creation as a single flow. Emit progress to stdout as each step completes so the user knows what's happening.

**New policy:**


Co-change lookup is opportunistic, not required. It is an enhancement on top of the baseline ReviewContext, not a hard dependency.

- If `REVIEW_CONTEXT_GITHUB_TOKEN` is missing OR GitHub API call fails for any reason (auth, rate limit, network) → skip co-change, continue assembly with baseline context, warn in provenance. Do not hard fail.


**The only remaining hard fail is:**
Missing or unresolvable Entire checkpoint trailer on the commit. That stays non-negotiable. Everything else is best-effort with transparent provenance.

1. `review-runner.ts` co-change lookup — wrap in try/catch, on any failure set `coChangeSkipped: true` and `coChangeSkipReason` in provenance instead of throwing. Assembly continues without related files.
